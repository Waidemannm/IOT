# 🌱 SmartGrass IoT

Sistema inteligente de monitoramento de gramado utilizando ESP32, sensores simulados, MQTT, Node-RED, OpenWeatherMap e Oracle Database.

---

## 🧠 Descrição do Projeto

O SmartGrass é uma solução IoT completa que monitora a umidade do solo e a altura da grama em tempo real, cruza essas informações com dados climáticos da cidade de São Paulo e decide automaticamente quando irrigar ou cortar o gramado.

A arquitetura é baseada no modelo **publish/subscribe** com protocolo MQTT, processamento no Node-RED e persistência em banco de dados Oracle.

---

## ⚙️ Tecnologias Utilizadas

| Tecnologia | Função |
|---|---|
| ESP32 (Wokwi) | Leitura dos sensores e publicação MQTT |
| HiveMQ Cloud | Broker MQTT com TLS |
| Node-RED | Processamento, dashboard e integração |
| OpenWeatherMap API | Dados climáticos em tempo real de São Paulo |
| Oracle Database (FIAP) | Persistência dos alertas gerados |
| Arduino / C++ | Firmware do ESP32 |
| JSON | Estrutura de dados trafegados |

---

## 🔌 Sensores Utilizados

### 📏 Sensor Ultrassônico (HC-SR04)
- Pinos: TRIG → GPIO 5 / ECHO → GPIO 18
- Mede a distância em cm
- Simula a altura da grama
- Valor normal: ≥ 399cm — abaixo disso indica grama alta

### 💧 Potenciômetro (Simulando Umidade do Solo)
- Pino: GPIO 34 (ADC)
- Simula leitura de umidade de 0% a 100%
- Abaixo de 30% indica solo seco

## 📡 Arquitetura do Sistema

```
┌─────────────────┐
│  ESP32 (Wokwi)  │  ← Sensores: Ultrassom + Potenciômetro
└────────┬────────┘
         │ MQTT TLS (porta 8883)
         ▼
┌─────────────────┐
│  HiveMQ Cloud   │  ← Broker MQTT na nuvem
└────────┬────────┘
         │
         ▼
┌─────────────────┐        ┌──────────────────────┐
│    Node-RED     │ ──────▶│  OpenWeatherMap API  │
└────────┬────────┘        └──────────────────────┘
         │
         ├──────▶ Dashboard (localhost:1880/ui)
         │
         └──────▶ Oracle Database (FIAP)
```

---

## 🔄 Funcionamento

1. O ESP32 lê os sensores a cada 700ms
2. Os dados são serializados em JSON e publicados via MQTT no HiveMQ Cloud
3. O Node-RED consome os tópicos `grama/sensor/#`
4. A cada 10 minutos o Node-RED consulta o clima atual de São Paulo
5. A regra de negócio cruza sensores + clima e gera um alerta
6. O dashboard exibe o alerta em tempo real
7. Cada alerta é salvo automaticamente no Oracle com timestamp

---

## 📦 Tópicos MQTT

| Tópico | Descrição |
|---|---|
| `grama/sensor/umidade` | Leitura da umidade do solo |
| `grama/sensor/ultrassom` | Leitura da distância (altura da grama) |

### Exemplo de payload

```json
{ "sensor": "umidade", "valor": 45.3 }
{ "sensor": "ultrassom", "valor": 238.0 }
```

---

## 🚦 Regras de Decisão

| Condição | Alerta |
|---|---|
| Distância < 399cm | ✂️ Hora de Cortar |
| Umidade < 30% | 🚿 Regar Agora |
| Ambos simultaneamente | Exibe os dois alertas |
| Tudo normal | ✅ Gramado OK |

## 🗄️ Banco de Dados

```sql
CREATE TABLE sensores (
    id   NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    tipo VARCHAR2(30) NOT NULL
);

CREATE TABLE alertas (
    id          NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    id_sensor   NUMBER NOT NULL,
    tipo_alerta VARCHAR2(10) NOT NULL,       -- 'OK' ou 'RUIM'
    descricao   VARCHAR2(255),
    data_hora   TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_alerta_sensor FOREIGN KEY (id_sensor) REFERENCES sensores(id),
    CONSTRAINT chk_tipo_alerta  CHECK (tipo_alerta IN ('OK', 'RUIM'))
);
```

## 📊 Dashboard

Acessível em `http://localhost:1880/ui`, exibe:

- Decisão inteligente com motivo (cortar / regar / ok)
- Clima atual de São Paulo (temperatura, chuva, vento, umidade do ar)
- Gauge de umidade do solo
- Gauge de distância (altura da grama)
- Gráfico histórico de umidade
- Gráfico histórico de distância
